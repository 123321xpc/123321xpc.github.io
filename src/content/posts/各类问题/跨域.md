## 什么是跨域？跨域有哪些解决方案？

+ 跨域是浏览器出于安全考虑，**阻止在一个域名上向另一个域名的服务器发送请求**的行为。
+ 跨域请求在以下情况会被阻止：
	1. 不同的协议（例如 HTTP 和 HTTPS 之间）。
	2. 不同的域名（例如 `example.com` 和 `another-example.com`）。
	3. 不同的端口（例如 `example.com:80` 和 `example.com:8080`）。

## 解决方案

### 1. **CORS (Cross-Origin Resource Sharing)**

- CORS 是一种规范化的方法，通过服务器在响应头中设置适当的 HTTP 头，浏览器可以允许某些跨域请求。
- 优点：支持各种 HTTP 请求方法（GET、POST、PUT、DELETE 等），安全性高。
- 缺点：需要服务器端的支持和配置。



###  2. **反向代理**

+ 反向代理是位于服务器端的一种代理服务器，它接收来自客户端的请求并将请求转发给后端的服务器（或多个服务器），然后将服务器的响应返回给客户端。客户端并不直接接触真实的服务器，而是与反向代理服务器进行交互。

- 使用反向代理服务器，将跨域请求代理到目标服务器。例如，使用` Nginx，devServer` 配置反向代理。
- 优点：透明化处理，客户端无需感知跨域。
- 缺点：需要额外的服务器配置。



**正向代理**：客户端通过正向代理服务器向目标服务器发送请求，然后代理服务器代表客户端向目标服务器请求资源，并将获取到的资源返回给客户端。

+ **使用场景**

	+ **隐藏客户端信息**：目标服务器只能看到代理服务器的请求，无法直接看到客户端的真实 IP 地址。

		**突破访问限制**：正向代理可以用于访问被限制的网络资源，例如翻墙访问某些网站。

		**缓存功能**：正向代理可以缓存服务器的响应数据，以提高访问速度。
